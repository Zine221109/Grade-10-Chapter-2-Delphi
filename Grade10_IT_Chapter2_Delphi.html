Data Types & Variables

Understanding what kinds of data your program can store and manipulate.

üóÉÔ∏è What is a Variable?

A variable is a named storage location in memory that holds a value. Think of it as a labelled box ‚Äî you put something in, you can take it out, and you can change what's inside. Variables must be declared before use in Delphi.

Delphi Pascal var
  iAge : Integer;          // Whole number
  fPrice : Real;           // Decimal number
  sName : String;           // Text
  bPassed : Boolean;        // True or False
  cGrade : Char;            // Single character
‚ö†Ô∏è
Naming Conventions (Important for tests!)

Use a prefix to show the data type: i for Integer, f for Float/Real, s for String, b for Boolean, c for Char. Use meaningful names: iAge not just x.

Everything from a TEdit comes in as a String. You cannot do maths on a String. You need to cast (convert) it to an Integer or Real first. Similarly, to display a number in a TLabel, you must convert it back to String.

üìù Worked Example: VAT Calculator

The user enters a price (String), we calculate VAT (15%), and display the result.

Delphi Pascal // Step 1: Declare variables
var
  fPrice, fVAT, fTotal : Real;
  sOutput : String;

begin
  // Step 2: Get input ‚Äî CAST from String to Real
  fPrice := StrToFloat(edtPrice.Text);

  // Step 3: Process / Calculate
  fVAT := fPrice * 0.15;
  fTotal := fPrice + fVAT;

  // Step 4: Output ‚Äî CAST back to String with 2 decimal places
  lblVAT.Caption := 'VAT: R' + FloatToStrF(fVAT, ffFixed, 8, 2);
  lblTotal.Caption := 'Total: R' + FloatToStrF(fTotal, ffFixed, 8, 2);
end;
‚ö†Ô∏è
Common Mistake!

If you forget to cast: lblResult.Caption := iAge + 5; ‚Äî this will cause a type mismatch error because you're trying to assign an Integer to a String property. Correct: lblResult.Caption := IntToStr(iAge + 5);

Events & Writing Code

How your program responds to user actions ‚Äî the heart of event-driven programming.

‚ö° What is an Event?

In Delphi, code does not run automatically from top to bottom. It runs in response to events ‚Äî actions performed by the user or the system. Each event has an event handler (a procedure/method) that contains the code to execute.

üñ±Ô∏è
OnClick

Fires when user clicks the component (button, checkbox, etc.)

üöÄ
OnCreate

Fires when the form is first created ‚Äî good for initialisation

‚è±Ô∏è
OnTimer

Fires repeatedly at set Interval (milliseconds) ‚Äî for animations & clocks

Conditional Constructs

Making decisions in your code ‚Äî IF statements, nested IFs, and CASE.

‚úÖ IF Statement (Basic)

Single IF:

if iAge >= 18 then
  lblResult.Caption := 'Adult';

IF-THEN-ELSE:

if iAge >= 18 then
  lblResult.Caption := 'Adult'
else
  lblResult.Caption := 'Minor';
‚ö†Ô∏è
IMPORTANT: No semicolon before ELSE!

In Delphi, the line before else must NOT have a semicolon. lblResult.Caption := 'Adult' ‚Üê no semicolon here!

üéì Worked Example: Grade Calculator

Delphi Pascal ‚Äî Nested IF procedure TForm1.btnGradeClick(Sender: TObject);
var
  iMark : Integer;
  sGrade : String;
begin
  iMark := StrToInt(edtMark.Text);

  if iMark >= 80 then
    sGrade := 'Outstanding (Level 7)'
  else if iMark >= 70 then
    sGrade := 'Meritorious (Level 6)'
  else if iMark >= 60 then
    sGrade := 'Substantial (Level 5)'
  else if iMark >= 50 then
    sGrade := 'Adequate (Level 4)'
  else if iMark >= 40 then
    sGrade := 'Moderate (Level 3)'
  else if iMark >= 30 then
    sGrade := 'Elementary (Level 2)'
  else
    sGrade := 'Not Achieved (Level 1)';

  lblGrade.Caption := sGrade;
end;

üì¶ BEGIN...END Blocks

When you need multiple statements in an IF branch, wrap them in begin...end:

if iMark >= 50 then
begin
  lblResult.Caption := 'Passed!';
  lblResult.Color := clGreen;
  ShowMessage('Congratulations!');
end
else
begin
  lblResult.Caption := 'Failed!';
  lblResult.Color := clRed;
end; // Note: semicolon AFTER end when not before else

üìã CASE Statement ‚Äî Multi-way Selection

Use case when comparing one variable against multiple specific values. Cleaner than many nested IFs!

Delphi Pascal ‚Äî CASE var iDay : Integer;
begin
  iDay := StrToInt(edtDay.Text);

  case iDay of
    1: lblDay.Caption := 'Monday';
    2: lblDay.Caption := 'Tuesday';
    3: lblDay.Caption := 'Wednesday';
    4: lblDay.Caption := 'Thursday';
    5: lblDay.Caption := 'Friday';
    6,7: lblDay.Caption := 'Weekend!'; // Multiple values
  else
    lblDay.Caption := 'Invalid day';
  end; // end of case
end;
üìå
When to use CASE vs IF?

Use CASE when comparing one variable against specific integer or char values. Use IF when you have range conditions (e.g., mark ‚â• 50) or string comparisons.
